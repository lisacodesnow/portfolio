Nametag Generator

Overview

The Name Tag Generator is an application that displays any name a user adds. The name tags are created using JSX and stateful components. Lifecycle methods, mounting, unmounting, and updating, are also used to save a name tag once the screen is refreshed or closed then opened.

Challenge

The challenge I had was using props effectively to manage data flow.

Solution

Understanding the component tree and breaking the project down into presentational and container components is how I learned to use props effectively to manage data flow.

A component tree is a hierarchy of React components that help to render UI.

First I decompose the UI and break the static image of the application into presentational and container components. While breaking down the components I also name them.

Presentational components handle rendering logic, how the UI looks. These components receive data in the form of props to show in JSX.

Container components are stateful components that handle and update the data model. Its concern is how data is used and passes props for the presentational components to render.

My UserInput and NameTagList UIâ€™s are going to change due to user interaction and will need data passed to them from the App component which updates and passes the data as props.

I simply see the stateful components as the parent container and presentational components as the children and even grandchildren containers.

Understanding how the application needs the data and seeing where it originates from makes passing props easier for me.
