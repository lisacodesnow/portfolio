Sticky Note

Overview

The Note Taking app is an interactive interface that allows users to create notes that are displayed on the screen. React is using JSX for the app's UI and stateful components to manage the data typed by the user.

Challenge

The challenge I had was making the application interactive, so it could display the user's typed words.

Solution

So far, I had built applications with functional components that used an already built data model. In order for notes, the user’s typed words, to be received, the state needed to have empty data models.

Utilizing event listeners and handlers is how I made the app interactive to display user’s typed words.

Event listeners listen for an event to happen, i.e. typing, and the event handler is triggered once the text is typed or button is hit. The event handler actually updates the state so the changes can be displayed on the screen.

Two event listeners and handlers were used:

  • The onChange listener works with the input element and listens for a change in value, i.e. entering text. Once the text is typed it calls the updateChalk event handler.
  • The onSubmit listener works with the form element and triggers the updateNotes event handler when pushing the submit button.

The two event handlers are functions that I created to update state:

  • The updateChalk event handler updates the “chalk” property empty string. This property is what is used to show the user's typed words.
  • The updateNotes event handler updates the “notes” property empty array. The array collects all the notes typed and submitted.

To accomplish app interactivity for users to experience is the rewarding part about coding.
